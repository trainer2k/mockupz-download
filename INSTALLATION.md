# Install Mockupz on macOS

Mockupz is distributed outside the Mac App Store through GitHub Releases. Follow these steps to install it safely.

## English

### 1. Download the official release

1. Open the **Releases** page of the official Mockupz GitHub repository.
2. Download the newest `Mockupz-…-macOS.dmg` or `.zip` file.
3. Do not download builds reposted by third parties.

### 2. Move Mockupz to Applications

- **DMG:** Open the disk image and drag **Mockupz.app** into **Applications**.
- **ZIP:** Double-click the ZIP file, then move **Mockupz.app** into **Applications**.

Start the installed copy from **Applications**, not directly from the Downloads folder or mounted DMG.

### 3. Allow the app to open

A normal macOS confirmation may appear because the app was downloaded from the internet. Click **Open**.

If macOS blocks the app or says that the developer cannot be verified:

1. Try to open **Mockupz** once and close the warning.
2. Open **Apple menu  → System Settings → Privacy & Security**.
3. Scroll to the **Security** section.
4. Click **Open Anyway** next to the Mockupz warning.
5. Authenticate with Touch ID or your Mac login password, then confirm **Open**.

On some macOS versions, you can also Control-click **Mockupz.app** in Applications, choose **Open**, and confirm **Open**.

Only bypass this warning when the file came from the official Mockupz GitHub Release.

### 4. Permissions

Mockupz does **not** need Screen Recording or Microphone access for its normal editing workflow.

When you import artwork, open a project, save a project or export a PNG, macOS may show a normal file or folder chooser. Mockupz receives access only to the locations and files you select.

### Troubleshooting

- **“Mockupz is damaged” or the app still will not open:** Delete that copy and download the newest official release again.
- **The Open Anyway button is missing:** Try opening Mockupz once more, then return immediately to Privacy & Security.
- **The app opens from the DMG but projects do not behave correctly:** Move the app to Applications and launch it from there.
- **After updating:** Replace the old app in Applications with the new version. Your saved project files remain separate from the app.

## Deutsch

### 1. Offizielle Version herunterladen

1. Öffne im offiziellen Mockupz-GitHub-Repository den Bereich **Releases**.
2. Lade die neueste Datei `Mockupz-…-macOS.dmg` oder `.zip` herunter.
3. Verwende keine von Drittanbietern erneut hochgeladenen Versionen.

### 2. Mockupz installieren

- **DMG:** Öffne das Image und ziehe **Mockupz.app** in den Ordner **Programme**.
- **ZIP:** Entpacke die ZIP-Datei und verschiebe **Mockupz.app** nach **Programme**.

Starte anschließend die installierte Version aus **Programme**.

### 3. Aus dem Internet geladene App erlauben

Beim ersten Start kann macOS darauf hinweisen, dass die App aus dem Internet geladen wurde. Bestätige mit **Öffnen**.

Falls macOS die App blockiert oder den Entwickler nicht verifizieren kann:

1. Versuche Mockupz einmal zu öffnen und schließe die Warnung.
2. Öffne **Apple-Menü  → Systemeinstellungen → Datenschutz & Sicherheit**.
3. Scrolle zum Bereich **Sicherheit**.
4. Klicke bei der Mockupz-Meldung auf **Dennoch öffnen**.
5. Bestätige mit Touch ID oder deinem Mac-Passwort und anschließend mit **Öffnen**.

Bei manchen macOS-Versionen funktioniert alternativ: Control-Klick beziehungsweise Rechtsklick auf **Mockupz.app** → **Öffnen** → nochmals **Öffnen**.

Umgehe diese Sicherheitsmeldung nur, wenn du Mockupz aus dem offiziellen GitHub Release geladen hast.

### 4. Benötigte Berechtigungen

Mockupz benötigt für die normale Nutzung **keine Bildschirmaufnahme- und keine Mikrofonberechtigung**.

Beim Importieren, Speichern oder Exportieren öffnet macOS einen Datei- oder Ordnerdialog. Die App erhält nur Zugriff auf die Dateien und Orte, die du selbst auswählst.
